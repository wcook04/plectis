# Microcosm Substrate

`microcosm-substrate/` is a clean public substrate prototype. It is a small, runnable root for the public-safe organs that should grow out of the macro system without copying its private state, stale public foldering, or egress-centered posture.

First command from a cold clone:

```bash
./bootstrap.sh --suite first-wave --emit receipts/cold_clone_probe.json
```

This seed intentionally implements the root contract, doctrine roof, atlas entry packet, receipt writer, private-state scan, and cold-clone probe before the Wave 1 organs land. Until those organs exist, the bootstrap command emits a typed `blocked` receipt naming the first missing organ or validator.

## Root Shape

- `AGENTS.md` is the local agent entry contract.
- `CONSTITUTION.md`, `AXIOMS.md`, `PRINCIPLES.md`, and `ANTI_PRINCIPLES.md` are the doctrine roof.
- `atlas/entry_packet.json` is the cold-start route surface.
- `core/` carries typed contracts, fixture manifests, acceptance plans, and private-state rules.
- `fixtures/first_wave/` carries public-safe synthetic fixtures only.
- `src/microcosm_core/` carries validators, receipts, and the bootstrap probe.
- `receipts/` is generated by commands and is not source authority.

## Current Claim Ceiling

This root proves that a cold clone can run a validator, emit receipts, and report missing organs with typed status. It does not prove the full macro system, any private data equivalence, or any external egress readiness.
