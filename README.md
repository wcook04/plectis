# Microcosm Substrate

This repository slice is a public substrate scaffold for a first-wave organ:
`pattern_binding_contract`.

The slice proves only these mechanics:

- a cold clone can import the package,
- synthetic fixture inputs can be scanned for non-public content bodies,
- pattern rows can bind to redacted source capsules,
- expected negative cases are observed with stable error codes,
- receipts are generated by commands instead of hand-authored as source.

It does not claim public runtime completeness, hosted readiness, publication
permission, or implementation of later organs.

## First Commands

From this directory:

```bash
python3 -m microcosm_core.validators.private_state_scan --root . --out receipts/first_wave/private_state_scan.json
python3 -m microcosm_core.organs.pattern_binding_contract validate --input fixtures/first_wave/pattern_binding_contract/input --out receipts/first_wave/pattern_binding_contract
./bootstrap.sh --suite first-wave --emit receipts/cold_clone_probe.json
```

## Boundary

The public substrate may carry public-safe projection contracts, schema-only
rows, synthetic replacement fixtures, redacted lineage, validators, and receipt
contracts. It must not carry forbidden content bodies, live operator state, raw
operator text, provider payload bodies, browser/HUD/cockpit state, recipient or
publication surfaces, prediction/market material, or old scratch-root contents
as source authority.

Existing files in this tree are target paths. The reconstruction authority lives
in the macro contracts that generated this slice, not in prior scratch content.
