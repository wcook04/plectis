# Axioms

1. JSON is contract; Markdown is projection.
2. Public fixtures may use public-safe macro bodies with provenance when a contract classifies the material and bounds the claims; true private material remains synthetic or omitted.
3. Registries, receipts, and handle counts are evidence handles, not completeness proof.
4. A validator pass proves the receipt boundary, not public runtime completion.
5. Expected negative cases are part of the runnable contract.
6. Generated receipts are not hand-authored source files.
7. Compact CLI cards are the first-reader path before raw receipt trees.
8. Public-safe copied bodies carry provenance and authority ceilings; private material is omitted or replaced with synthetic fixtures.
