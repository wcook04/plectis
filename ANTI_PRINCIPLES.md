# Anti-Principles

Microcosm explicitly rejects these failure attractors. Each row names the
axioms whose floor it violates; the row is a guardrail, not an independent
source of authority.

| ID | Violated axioms | Rejected failure shape |
| --- | --- | --- |
| AP-1 Fixture-label echo | AX-1, AX-12 | Echoing a fixture verdict, registry label, markdown claim, or model answer instead of recomputing it from lower-level evidence. |
| AP-2 Producer trust | AX-2, AX-3 | Trusting a producer, persona, route, credential, or prior success instead of a checker, proof, policy, or dereferenced capability. |
| AP-3 Rank-as-product-score | AX-5, AX-12 | Treating counts, handles, passing validators, or evidence classes as product maturity, release readiness, or whole-system correctness. |
| AP-4 Cache-across-drift | AX-4, AX-10 | Caching by name, path, route, display label, or producer while basis content has drifted. |
| AP-5 Unknown-unknown exhaustiveness | AX-6 | Treating missing evidence as evidence of absence outside a declared closed domain. |
| AP-6 Inadmissible number emission | AX-7 | Emitting inadmissible numbers or conclusions instead of typed refusal. |
| AP-7 Public/private membrane breach | AX-8 | Letting untrusted, private, provider, prompt, credential, or raw-voice material reach a privileged or public sink without provenance and sanitizer transform. |
| AP-8 Blind irreversible mutation | AX-9 | Performing irreversible mutation without a compensator, rollback receipt, CAS/world-version guard, or explicit irreversible boundary. |
| AP-9 Frozen live fact | AX-10 | Freezing live facts in prose without `as_of`, basis, and rederive contract. |
| AP-10 Prose-as-executable-authority | AX-11 | Granting executable authority to prose doctrine without grammar and receipt obligations. |
| AP-11 Meta-artifact exemption | AX-12 | Exempting Microcosm's own receipts, docs, releases, generated projections, or validators from Microcosm's evidence gates. |
| AP-12 Synthetic substrate substitution | AX-1, AX-4, AX-8 | Using synthetic fixtures as substitutes for available public-safe substrate. |
| AP-13 Generated-receipt source inversion | AX-4, AX-11 | Treating generated receipts as hand-authored source. |
| AP-14 Release-authority inflation | AX-5, AX-8, AX-12 | Letting public docs imply hosted release, provider-call authority, source-mutation authority, financial advice, product readiness, affiliation, or whole private-system release. |
| AP-15 Mechanism theater | AX-1, AX-2, AX-12 | Using field vocabulary such as proof, verifier, information flow, world model, or benchmark integrity when the public mechanism only validates a receipt or projection shape. |
| AP-16 Receiver inflation | AX-8, AX-12 | Displaying a declared, inferred, or generated downstream target as an observed runtime effect without evidence of the effect relation. |
| AP-17 Projection-as-source | AX-4, AX-11 | Letting a markdown page, generated card, atlas row, or compact receipt summary override the executable contract it summarizes. |
