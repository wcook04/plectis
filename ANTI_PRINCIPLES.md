# Anti-Principles

These are explicit failure shapes the public Microcosm rejects.

- A projection row claims to be source authority.
- Metadata-only stand-ins replace public-safe source that could be copied with
  provenance and bounded claims.
- A synthetic fixture pretends to be real substrate.
- Secret, credential, session, provider payload, raw operator voice, or private
  personal material is exported.
- A passing validator, accepted organ count, handle count, or legibility score
  claims release readiness, product correctness, or whole-system completeness.
- A public-safe macro body is forced into synthetic reimplementation solely
  because it is body-bearing.
- A Lean/proof body gets a special exemption or special ban instead of using the
  same public-safe import classification as tools, patterns, and receipts.
- Duplicate pattern ids are accepted without deterministic rejection.
- An unsupported authority-chain handle is treated as authority.
- A generated receipt is edited by hand.
- Raw receipt trees are the first required entry path when compact cards can
  answer the reader's initial question.
- Root docs imply hosted publishing, provider-call, source-mutation, financial
  advice, release, or production authority.
