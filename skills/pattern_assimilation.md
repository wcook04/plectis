# Pattern Assimilation Skill

Use this skill after a local organ, validator, fixture, or receipt teaches a reusable substrate rule.

## Procedure

1. Name the local lesson in one sentence.
2. Identify the owning plane: standard, organ, validator, fixture, atlas, or bootstrap.
3. Decide whether the lesson changes behavior. If yes, patch the owning plane. If no, emit a `nothing_to_refine` receipt.
4. Record the evidence command and receipt path.
5. Keep the lesson public-safe: source classes and pattern IDs are allowed; private source bodies are not.

## Seed Lesson

The public root should bootstrap before every Wave 1 organ exists. A typed `blocked` receipt is better than a green checklist because it tells the next seed the first missing organ, exact missing path, and unblock condition.
