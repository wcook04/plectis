# Microcosm Work Item Author

Use this public-safe skill projection when a Microcosm work item needs governed admission as a durable unit of substrate work with state, dependencies, evidence refs, closure rules, and public/private boundaries. It is not a route for treating a chat todo, release checklist, or live private task row as source authority.

## Authority Boundary

This skill source governs work-item admission only. A work item can identify and route substrate work; it does not prove execution, validation, release readiness, private-data equivalence, or downstream projection freshness until the owning receipts and checks settle those claims.

Anti-claims:

- A chat todo, release checklist, generated queue, or live private task row is not work-item source authority.
- Work-item admission does not prove the work is complete, validated, public-safe, or release-ready.
- Work-item triad resolution must not weaken std_microcosm_work_item or convert state labels into execution proof.
