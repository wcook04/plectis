## Anti-patterns

- **Consume-only closeout.** Using a general artifact locally, then ending the turn without asking whether it should be refined.
- **Complaint-as-alias patch.** The operator names a systemic failure class ("hidden knowledge", "correct guessing", "tribal knowledge", "make it accessible without bloat") and the agent responds by adding those literal words to docs-route or skill triggers without proving that route discoverability was the underlying defect. This preserves the guesswork under a new synonym. The correct response is to locate the actual implicit dependency and repair that source; route aliases are only valid when the route miss itself is demonstrated.
- **Apply-lane avoidance for principle work.** The operator asks for a principle or the local case is clearly principle-grade, and the agent refuses to author it because the graph has a "curation/apply" path. The correction is not to bypass governance; it is to use the governance directly: inspect the existing graph, append or refine the complete `pri_*` row, update the owning standard/skill if relevant, and verify the principle surface.
- **Adapter bloat.** Solving discoverability by pasting a full workflow into `CLAUDE.md` or `CODEX.md` instead of linking to the skill and paper-module roof.
- **Wrong plane home.** Putting procedural learning into raw seed, or a principle-grade commitment into a skill body, because it is convenient in the moment.
- **Session-only learning.** Explaining in the final response what the system learned, but not depositing that learning into any durable plane artifact.
- **Treating one local case as the whole rule.** The general artifact should converge, not be rewritten as if one turn were representative of everything.
- **Dirty-tree monolith.** Treating raw seed, ledgers, generated projections, source edits, staged work, and reconstructable build caches as one generic dirty-tree blocker. The correction is a typed landing diagnosis: ambient operator source is excluded unless owned, tracked cache output is quarantined and repaired, owned source commits through scoped tooling, and genuine overlap blocks only the overlapping path set.
- **Above-and-beyond as either scope creep or tokenism.** Treating care or passion as permission for unbounded wandering is wrong, but treating it as "one safest tiny refinement and stop" is the mirror failure. The valid move is maximum coherent verified progress inside the trust envelope, with explicit evidence for what moved and a named blocker only where the stronger wave is genuinely unsafe or unavailable.
- **Null-pass laundering.** Treating a second `nothing_to_refine`, `already_settled`, `no clean high-value lane`, or CAP-only closeout as valid because the narrow requested lane is exhausted. The valid move is to broaden to the next bounded substrate-care lane or report the exact owner/policy/claim blocker plus re-entry condition.
- **Settlement-as-refinement.** Treating generated-state, Task Ledger, Work Ledger, validation, or clean git status as the same thing as refinement. Settlement may be complete while stewardship still finds a missing source bundle, audit sidecar, projection-tail contract gap, or operator-friction pattern.
- **Private-state humility as web-research refusal.** A bridge model sees pasted private evidence and says "this is private, so I will not research externally," then emits a neat synthesis from the paste alone even though public repos, docs, papers, specs, or current web facts would improve the answer. The valid move is lane separation: private repo facts via Type A evidence requests; public-world grounding via online search with citations.
- **Help-flag-only launch.** Adding a command flag and tests, but not updating the opening ladder or entry surface that cold agents actually consult.
- **Open flag after self-resolution.** Authoring a §backlog / §Refinement-log / §Anti-patterns flag earlier in the same turn (or immediately prior turn), then fixing what the flag pointed at, then yielding without closing the flag. The flag becomes upstream-prose drift pointing at resolved substrate; a future harvest cannot tell whether to trust the flag or treat the substrate as fixed. Same-turn flag closure (Workflow §5 sub-rule) closes this loop. Pair with PD3 §sharpening 3 — together they govern both halves: NAME drift when other agents are mid-mutation; CLOSE flags when you authored them and resolved them yourself.
- **First-contact plane assumed-equivalent.** Treating "the artifact exists at its canonical path" or "exact-slug `--ids` lookup resolves" as proof of first-contact discoverability. Each first-contact plane has its own selector machinery — `--docs-route` reads keyword/intent matches in `documentation_theory_index.json::machine_routes`; `--entry` runs a Python predicate chain in `comprehension_snapshot.py`; `--context-pack` runs anchor predicates with budget-trim protection in `navigation_context_pack.py`. A cold agent typing the operator's natural phrase exercises all three independently. Closure for a discoverability cap requires checking each plane that a future cold reader might enter through, then patching the right selector owner per plane — not adding more tokens to the plane already fixed. (Earned 2026-05-09 in the page-meta multi-plane closure session.)
- **Plane-owner mismatch — adding tokens to the fixed plane.** When one route plane resolves but another still fails, do not pile more tokens, aliases, or keywords onto the plane that already works. Read the failing plane's selector source (Python classifier order, anchor predicates, embedding seeds) and find the actual selector owner. The page-meta session decisive move was discovering that `--entry` failure was governed by Python predicate ordering (`is_navigation` ran before `is_frontend_capability_discovery`), not by docs-route registry tokens. Adding more docs-route tokens would have churned without effect.
- **Closeout claim trusted from prose, not verified against HEAD.** When a closeout summary claims "the live-state assertion was removed" or "the broken section was rewritten" and that claim controls cap retirement, commit safety, or downstream agent decisions, verify against actual `HEAD` content before treating the cap as closed. Use `git show <commit> -- <path>` to inspect the actual `+`/`-` diff, or `grep`/`Read` against the current file to confirm the claimed state. A reviewer noticing a possible contradiction is a tripwire to switch from prose-trust to disk-truth, not a moment to re-explain the prose. Diff/prose summaries are not enough when the claim drives closure.
- **Live-state test fragility.** Tests asserting that current `state/task_ledger/views/*` (or any live mutable substrate view) contains specific row content — "any sample item has dependency edges", "at least one bucket has WIP rows", "the third sample item is `td_X`" — will flap across concurrent agent activity. Pin live-surface tests to schema/topology only (bucket-set, per-row field presence, type-shape via `isinstance`). Pin row-content behavior with deterministic unit tests against synthetic input feeding the same compaction/transform helpers directly. The page-meta wave's Vantage test fix split this way: live test pins the work_spine bucket set + dependency_status field shape; sibling unit test pins `_compact_task_ledger_dependency_edges` against synthetic edges with known IDs. (Earned 2026-05-09 when a Vantage assertion required today's Task Ledger samples to contain dependency edges and broke under concurrent agent activity.)
