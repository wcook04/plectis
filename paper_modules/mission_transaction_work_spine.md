# Mission Transaction Work Spine

This source-available module validates the public mission transaction work-spine over the imported `work_landing` substrate.

The slice consumes `microcosm_core.macro_tools.work_landing` receipts for public work landing status, attempt binding, and source-faithful 12-step reconcile ordering, then layers dependency unlock gates, same-path claim conflicts, expected-parent checks, scoped mutation ceilings, exact receipt drains, closeout status projections, and schedulability decisions over bounded fixture rows.

Synthetic task, claim, checkpoint, and dependency rows remain regression and negative-case harnesses. They are not product evidence unless the organ binds them to body-free public work-landing receipts, secret-exclusion scan output, provenance refs, and runtime validation.

It does not mutate live work ledgers, git state, Task Ledger authority, raw operator material, provider material, release surfaces, or private source bodies.
