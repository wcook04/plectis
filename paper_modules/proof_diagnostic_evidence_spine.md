# Proof Diagnostic Evidence Spine

## Teleology

`proof_diagnostic_evidence_spine` is the body-safe evidence membrane before
formal proof work. It records proof/evidence diagnostics while rejecting proof
bodies, provider output bodies, source-authority upgrades, stale coupling, and
runtime-correctness overclaims.

## Public Contract

The validator consumes Ring2 failure-taxonomy, graph-update, verifier-trace
repair, and formal evidence-cell anchor receipt refs and emits diagnostic
receipts over those refs. Provider-advisory rows are not proof authority.
Passing diagnostic checks do not become formal proof authority or theorem
correctness.

## Receipt Expectations

The command emits proof receipts, provider payload policy results, diagnostic
board output, proof-evidence validation, and fixture acceptance receipts.

## Anti-Claim

This module documents diagnostic receipt anchors over real Ring2 substrate and
keeps forbidden proof/provider body cases as regression-only guards. It does not
run Lean, call providers, expose proof bodies, prove runtime correctness,
certify public release operations, authorize publication or recipient work,
establish secret export, or claim whole-system correctness.
