# Plectis Public Site Source Contract

This site is a static public projection of safe Plectis source material.
It is not a runtime window into the private working environment and it is not
a second source of truth. The standalone public repository remains the
source-of-record object for the public slice: Type A agents should read
`AGENTS.md`, `README.md`, `QUICKSTART.md`, and the local result records, then
answer from the repository, this site, and the AI review packet.

## Contract

- The public repository is the source of record for the standalone released
  slice. The private working root remains out of scope.
- The Type A agent-entry surface is the standalone `microcosm-substrate`
  repository/export, starting from `AGENTS.md`, `README.md`, `QUICKSTART.md`,
  `atlas/entry_packet.json`, and the local `microcosm` CLI. These compatibility
  identifiers stay lower-case until a dedicated Plectis repo and route matrix exist.
- The website is a presentation and documentation map dynamically tied to
  public source refs. Website, launch-page, and public-projection changes route
  through dissemination/public-export owners, then link back to the repository.
- `sites/microcosm/index.html` is curated manually for product judgment.
- `sites/microcosm/docs/`, `content-manifest.json`, and deployment envelope
  files are generated by `tools/meta/dissemination/build_microcosm_public_site.py`.
- `sites/microcosm/docs/architecture-graph-scene.json` is generated by the
  same builder from public source refs and is the machine-readable packet behind
  the architecture map; it is a projection, not source authority.
- Generated pages read only public Plectis source refs under the compatibility
  `microcosm-substrate/`; material outside the standalone release remains
  private and unlicensed by this site.
- Source-derived strings are HTML-escaped; raw HTML from source markdown is not
  executed.
- The committed site has no runtime backend, no cookies, no analytics, no
  forms, no external data fetches, and no browser path into private state.
- Deployment headers must keep `connect-src 'none'` until a concrete public
  feature requires a connection and the exception is reviewed.
- HSTS is deferred until the final HTTPS domain is verified. When `--base-url`
  is supplied, the builder emits host-only HSTS by default. `--hsts-mode
  include-subdomains` and `--hsts-mode preload-consented` require explicit
  domain inventory / preload consent evidence before use.

## Editorial voice

Two voices live on this site, and only one is open to free rewriting.

- **Site voice** — `index.html` and the builder's own editorial strings (page ledes,
  section intros, callouts, the area descriptions in `family_page_info()`, nav labels,
  page titles, the `404` copy). Rewrite these for clarity and tone. Lead with what runs
  and what it shows; name the evidence and the boundary; and do not use proof,
  certificate, or authority language unless the named checker actually computes it.
  Prefer plain active sentences; use em dashes sparingly.
- **Source-derived voice** — the component `Does` / `Does not prove` / job text, the
  evidence-class member lists, and the paper-module bodies, all projected from
  `microcosm-substrate/`. Do not rewrite these on the site: that diverges the projection
  from its source and breaks the source discipline above. Fix them at the
  substrate source, then reproject.

The validation floor below checks private leakage, banned vocabulary, internal links,
and drift. It does not check punctuation, overclaiming, vague verbs, or repeated
phrasing, which stay an editorial judgement call.

## Validation Floor

Before landing site changes, run:

```bash
./repo-python tools/meta/dissemination/build_microcosm_public_site.py --write --validate
./repo-python tools/meta/dissemination/build_microcosm_public_site.py --check --validate
```

Before public release, verify the deployed host response envelope, not only the
committed static files:

```bash
./repo-python tools/meta/dissemination/verify_microcosm_public_site_host.py --base-url https://<public-domain> --require-hsts
```

Use `--require-hsts-preload` only after the final domain and all covered
subdomains are intentionally inside the preload boundary. The host verifier also
requires `security.txt` to link to a same-origin vulnerability disclosure policy
page (`docs/vulnerability-disclosure.html`); a generic source or license page is
not enough for release signoff.

For served-effect proof, serve `sites/microcosm/` locally and check route
status, console errors, headers, mobile drawer behavior, and internal links.
