# Release Discipline

These rules govern the public Microcosm boundary. They are corollaries of the
axioms, not axioms themselves.

1. JSON contracts, registries, validators, receipts, and source manifests are
   authority surfaces. Markdown and public site pages are projections.
2. Public fixtures may use public-safe copied macro bodies when a contract
   classifies the material, proves provenance, and bounds the claims.
3. True private material is synthetic, redacted, or omitted.
4. Registries, receipts, scanner results, and handle counts are evidence
   handles, not completeness proof.
5. A validator pass proves only the named command, input boundary, and contract.
6. Expected negative cases are part of the runnable contract.
7. Generated receipts are never hand-authored source.
8. Compact CLI cards are first-reader routing surfaces before raw receipt trees.
9. Public-safe copied bodies carry provenance and authority ceilings.
10. Release, hosted-product, provider-execution, source-mutation,
    financial-advice, affiliation, or whole-private-system claims require
    explicit release authority and fail closed by default.
